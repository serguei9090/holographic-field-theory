## 🛠️ 2. The Inference Normalization Mismatch Fix

### The Problem:
Initially, text generation outputted a random jumble of words (e.g. `Once upon a time story laughing any strange faceaws Gem mooacy...`) despite having low perplexity. 

We discovered a **distribution shift mismatch** between training and inference:
* **Training & Evaluation**: Context vectors $\psi$ were normalized using **Complex LayerNorm** and divided by $\sqrt{D}$.
* **Generation**: The iterative Hopfield attractor query (`refine_query`) was normalizing the vector using **L2 Norm** (`refined / norm`). 

### The Solution:
We updated `refine_query` and `query_topk` in [model.py](file:///i:/01-Master_Code/Test-Labs/01-CHFT/colab/src/model.py) to accept an optional normalization function `ln_fn`. In [generate.py](file:///i:/01-Master_Code/Test-Labs/01-CHFT/colab/src/generate.py), we passed the codebook's Complex LayerNorm (`codebook.ln`) into the Hopfield query. This aligned training and inference perfectly, raising the generation diversity to **70.7%** and producing actual, readable English.

## 🔄 4. Code Improvements for Next Run

1. **Data Leakage Fix**:
   We noticed that the random shuffle in `prepare_dataset` was unseeded. Resuming training on a new day created a new train/val split, contaminating the validation set with previously seen training data. We added a seeded generator to [train.py](file:///i:/01-Master_Code/Test-Labs/01-CHFT/colab/src/train.py):
   ```python
   g = torch.Generator().manual_seed(42)
   perm = torch.randperm(num_total, generator=g)
   ```
2. **Early Stopping & Best Checkpoint Save**:
   We added early stopping with `patience = 2`. The training script will now automatically detect when validation loss degrades, halt training, and save the best epoch weights to `best_chft_checkpoint.pth`.
